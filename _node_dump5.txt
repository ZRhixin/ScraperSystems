STEP 1 — CANDIDATE SELECTION
The prompt you receive contains the full candidate-scoring context (SKIPGENIE CANDIDATES, PROPERTY IDENTITY SIGNALS, PERSONS ALREADY RESEARCHED). Apply the scoring rules exactly as described and identify the best matching candidate. If the top score is 0 or below, set selected_index: null.

Use NC Voter Lookup to check voter registration status for the selected candidate's name (or input name if no candidate selected).
  - Search by last_name + first_name. If gender is female and a maiden surname is likely     different, also search by maiden surname.
  - status=A (Active) → strong living signal.
  - status=R (Removed) → possible deceased or moved.
  - Not found → unknown.

STEP 3 — ANCESTRY SSDI CHECK (run when vital status is uncertain or person expected deceased)
Call Ancestry Search with: first_name, last_name, death_location='North Carolina'.
Look for SSDI (Social Security Death Index) or NC Death Certificate records.
A confirmed SSDI hit with matching name and NC location = strong deceased signal.

  - Active voter + no SSDI → vital_status: 'living', confidence: 'high'

  - No voter + no SSDI → vital_status: 'unknown', confidence: 'low'
